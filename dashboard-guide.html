Naves Scraper / Guía del dashboard

Cómo usar el dashboard

Guía completa para gestionar el scraper de naves industriales desde el navegador. No se requieren conocimientos técnicos.

Acceso al dashboard
info Abre http://localhost:8501 en el navegador del equipo donde están instalados los servicios. Introduce la contraseña que configuraste en DASHBOARD_PASSWORD durante la instalación.
warning El dashboard requiere que la API esté en marcha (bash run_api.sh). Si ves el mensaje "No se puede conectar a la API", arranca el servicio primero.
Navegacion — barra lateral izquierda

El panel lateral muestra las seis secciones del dashboard. Haz clic en cualquiera para navegar. El botón Cerrar sesion en la parte inferior cierra la sesión del dashboard (no afecta al scraper).

Resumen
Visión general del sistema de un vistazo
expand_more

La página de inicio muestra el estado actual de todo el sistema en cuatro métricas clave y dos paneles de detalle.

  • counter_1 Total anuncios — cuántos anuncios hay en la base de datos local.
  • counter_2 Sincronizados en Webflow — anuncios ya publicados en el CMS.
  • counter_3 Pendientes de sync — anuncios nuevos que aún no se han enviado a Webflow.
  • counter_4 Estado del scraper — si está activo, inactivo, con error o pausado por captcha.

Si el scraper está en ejecución, la página se actualiza automáticamente cada 5 segundos para mostrar el progreso en tiempo real.

check_circle Si ves "Inactivo y listo para ejecutar", el sistema funciona con normalidad y puedes lanzar el scraper desde la sección Control.

Control del scraper
Iniciar, detener y renovar la sesión manualmente
expand_more

Estado actual — muestra si el scraper está corriendo, parado o si hay un error. Si hay un captcha activo, aparece un aviso naranja con instrucciones.

circleInactivo syncEn ejecución warningCaptcha activo errorError check_circleDetenido OK

Opciones de ejecución:

  • settings Máximo de páginas (0 = sin límite) — limita cuántas páginas de resultados escanea. Útil para pruebas o actualizaciones rápidas.
  • science Modo prueba — ejecuta el scraper sin guardar nada en la base de datos. Útil para verificar que la sesión funciona.
  • restart_alt Empezar desde la primera página — ignora el punto de parada anterior y vuelve a escanear desde el principio.

Pulsa Iniciar scraper para lanzarlo. El botón se desactiva mientras el scraper está en ejecución.

Pulsa Detener scraper para pararlo de forma segura. El scraper guarda el progreso antes de cerrarse.

Renovación de sesión — si Milanuncios bloquea el scraper por sesión caducada, usa este botón para abrir Chrome en el servidor y volver a iniciar sesión.

refresh Cómo renovar la sesión
  1. Asegúrate de que el scraper está detenido.
  2. Haz clic en "Abrir Chrome para renovar sesión".
  3. Ve al escritorio del servidor y busca la ventana de Chrome.
  4. Si aparece un captcha, resuélvelo primero.
  5. Inicia sesión con tu cuenta de Milanuncios.
  6. Navega a Mis Anuncios — la sesión se guardará automáticamente.
  7. El dashboard mostrará una notificación cuando la sesión se haya guardado.
schedule
Programacion
Configurar cuándo se ejecuta el scraper automáticamente
expand_more

Define el horario de ejecución automática del scraper. Una vez configurado, el scraper arrancará solo sin necesidad de intervención.

Próxima ejecución programada — aparece en la parte superior indicando la fecha y hora exacta del próximo arranque automático.

Horarios predefinidos disponibles:

  • scheduleTodos los días a las 6:00 (recomendado)
  • scheduleDos veces al día (6h y 18h)
  • scheduleDías laborables a las 7:00
  • scheduleCada 4 horas
  • scheduleCada hora
  • blockSolo manual (sin programación)

También puedes escribir tu propia expresión cron en el campo de texto si necesitas un horario personalizado.

El campo Máximo de páginas en ejecución programada limita el escaneo en las ejecuciones automáticas. Ponlo a 0 para sin límite.

Pulsa Guardar configuración para aplicar los cambios. El horario se activa inmediatamente sin necesidad de reiniciar nada.

Registros
Ver la actividad del scraper en tiempo real
expand_more

Muestra las últimas líneas del archivo de registro del scraper. Útil para diagnosticar problemas o ver qué está haciendo el scraper.

  • tune Número de líneas a mostrar — ajusta el slider para ver más o menos historial (entre 50 y 1000 líneas).
  • refresh Actualizar cada 5 segundos — activa esta casilla mientras el scraper está en marcha para ver el progreso en tiempo real.
info Busca líneas con [Listing] OK para ver anuncios procesados, o [Search] Página N para el progreso de paginación.
warehouse
Anuncios
Explorar y filtrar los anuncios obtenidos
expand_more

Muestra todos los anuncios de naves industriales almacenados en la base de datos local, con paginación de 50 por página.

Filtros disponibles (haz clic en "Filtros de búsqueda" para expandirlos):

  • location_onProvincia — filtra por nombre de provincia (ej. "Madrid").
  • straightenSuperficie mínima (m²) — muestra solo naves a partir de ese tamaño.
  • euroPrecio máximo — filtra por precio de venta o alquiler.
  • sortOrdenar por — fecha de obtención, precio, superficie o fecha de publicación.

Columnas de la tabla:

  • tagID — identificador único del anuncio en Milanuncios.
  • titleTitulo — título del anuncio.
  • mapProvincia / Localidad — ubicación.
  • euroPrecio / Euros por m² — precio total y ratio por metro cuadrado.
  • straightenSuperficie (m²).
  • categoryTipo — venta o alquiler.
  • cloud_doneID en Webflow — si está relleno, el anuncio ya está en el CMS.

Usa los botones Anterior y Siguiente al pie de la tabla para navegar entre páginas de resultados.

cloud_sync
Webflow
Sincronizar anuncios con el CMS de la web
expand_more

Gestiona la sincronización de los anuncios locales con Webflow CMS, donde aparecerán publicados en la web.

Métricas:

  • databaseTotal anuncios — todos los anuncios en la base de datos local.
  • cloud_doneSincronizados — ya enviados a Webflow.
  • cloud_uploadPendientes — nuevos anuncios listos para sincronizar.

Si hay anuncios pendientes, aparece el botón Sincronizar N anuncio(s) con Webflow. Al pulsarlo, el proceso:

  1. Sube las imágenes de cada anuncio a Webflow.
  2. Crea la ficha del anuncio con todos los datos.
  3. Lo crea como borrador en Webflow (no publicado automáticamente).
  4. Marca el anuncio como sincronizado en la base de datos local.
warning Los anuncios se crean como borradores en Webflow. Para publicarlos en la web debes entrar al editor de Webflow y publicarlos manualmente.
Situaciones frecuentes
warning
Captcha detectado en Chrome
El scraper espera — resuelve el desafío en la ventana de Chrome
expand_more

Cuando Milanuncios muestra un captcha interactivo durante el scraping, el scraper no se cierra — pausa y espera hasta 10 minutos para que lo resuelvas manualmente.

El dashboard muestra un aviso naranja en la parte superior de todas las páginas con instrucciones.

check_circle Cómo resolverlo
  1. El dashboard mostrará "Captcha detectado en Chrome" en naranja.
  2. Ve al escritorio del servidor y busca la ventana de Chrome (debería estar visible).
  3. Resuelve el captcha tal como aparece en pantalla (imágenes, puzzle, etc.).
  4. El scraper detectará que se resolvió y continuará automáticamente.
  5. El aviso naranja desaparecerá del dashboard en los próximos 5 segundos.
timer El scraper espera un máximo de 10 minutos. Si no se resuelve en ese tiempo, para automáticamente y el dashboard pedirá renovar la sesión.
lock
Sesion bloqueada — renovar login
Las cookies han caducado o Milanuncios ha bloqueado el acceso
expand_more

Las cookies de sesión de Milanuncios duran aproximadamente 30 minutos de inactividad. Cuando caducan, el scraper para automáticamente y el dashboard muestra un aviso rojo.

refresh Cómo renovar la sesión
  1. Accede a la sección Control del scraper.
  2. Asegúrate de que el scraper está detenido. Si no, pulsa Detener scraper primero.
  3. En la sección Renovación de sesión, pulsa "Abrir Chrome para renovar sesión".
  4. Ve al escritorio del servidor — Chrome se habrá abierto.
  5. Si aparece un captcha, resuélvelo primero.
  6. Inicia sesión en Milanuncios con tu cuenta.
  7. Navega a Mis Anuncios — el sistema guardará la sesión automáticamente.
  8. Aparecerá una notificación en el dashboard confirmando que la sesión se renovó.
  9. Vuelve a iniciar el scraper normalmente.
wifi_off
No se puede conectar a la API
El servicio de backend no está en marcha
expand_more

Si el dashboard muestra el error "No se puede conectar a la API" en rojo, significa que el servicio FastAPI no está corriendo.

terminal Solución
  1. Abre un terminal en el servidor.
  2. Ve a la carpeta del proyecto: cd naves-scraper
  3. Arranca la API: bash run_api.sh
  4. Espera 5-10 segundos y recarga el dashboard.

Si quieres que la API arranque en segundo plano (sobrevive al cierre del terminal):

nohup bash run_api.sh > logs/api.log 2>&1 &
echo $! > logs/api.pid
error
El scraper muestra estado "Error"
El proceso terminó de forma inesperada
expand_more

El estado Error en rojo indica que el scraper se cerró de forma inesperada. El mensaje de error aparece junto al estado.

  • article Ve a Registros para ver las últimas líneas del log y entender qué ocurrió.
  • lock_open Si el error menciona sesión o login, renueva la sesión desde Control del scraper.
  • play_circle Una vez identificado el problema, vuelve a lanzar el scraper normalmente.
info El scraper guarda su progreso automáticamente. Al relanzarlo, retomará desde el último punto de parada sin repetir trabajo ya hecho.
Flujo de trabajo habitual
check_circle El scraper corre automáticamente cada día a las 6:00 (configurable en Programación). En condiciones normales no necesitas hacer nada — solo revisar el Resumen de vez en cuando.
event_repeat Ciclo diario típico
  1. A las 6:00 el scraper arranca automáticamente y empieza a escanear Milanuncios.
  2. Los nuevos anuncios se guardan en la base de datos local.
  3. El scraper para cuando encuentra anuncios ya conocidos (modo incremental).
  4. Accede al dashboard → Webflow → pulsa Sincronizar para enviar los nuevos anuncios al CMS.
  5. Entra al editor de Webflow y publica los borradores nuevos para que aparezcan en la web.
